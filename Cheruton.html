Situation

While playing various games, one question that has often come across my teams mind is how these games were created and designed. So we took a school program as an opportunity to learn Game Development. Inspired by some of our favourite childhood games such as Maplestory, we built our own original 2D metroidvania.

  • GitHub
  • YouTube

What I did

I designed and implemented most player-facing features. For the character controller, gamefeel was very important to us. We made a lot of UX improvements including, Input buffering and Various movement helps (coyote time, platform boosts, corner correction, etc.). For game mechanics extensions, I implemented a lot of physically-based mechanics such as rope swinging, and IK using FABRIK on some of the monsters. For the software engineering side, I manage to pick up many useful patterns from reading the source code of other games. These include heavy use of state machines, observer patterns and dependency injection.


We wanted to explore drawing as art is what many people look for when searching for a game. 2D pixel art has been very popular among Indie developers because of its low barrier to entry. In this pursuit, we looked at various art resources to pick up the basics. In the end we used Aseprite for original asset creation.



Lastly, I implemented a basic dialogue system and made up some stories with the basic writing skills I had.
